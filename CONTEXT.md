# RoastLog

自宅でコーヒーを焙煎する個人ユーザーが焙煎記録を管理・分析するアプリケーションのドメイン言語。

## Language

### 焙煎の記録

**RoastLog（焙煎ログ）**:
1回の物理的な焙煎バッチを記録したエンティティ。同じ豆を連続して焼いた場合も、バッチごとに独立した RoastLog として記録する。
_Avoid_: 焙煎セッション、焙煎記録

**Batch（バッチ）**:
焙煎機に1回投入して焼き上げる単位。1 Batch = 1 RoastLog。
_Avoid_: ロット、セッション

**RoastDuration（焙煎時間）**:
豆投入から取り出しまでの総時間（秒）。冷却時間は含まない。0秒を投入時点とし、FirstCrack・SecondCrack のタイミングも同じ基点で計測する。

**ProcessNote（焙煎メモ）**:
焙煎後に記録する自由テキスト。「ハゼが予想より早かった」「途中で温度を上げた」など実際に起きたことのメモ。計画や事前メモは想定しない。
_Avoid_: 焙煎プロセスメモ、手順メモ

**WeightLossRate（重量減少率）**:
`(1 − 焙煎後重量 / 焙煎前重量) × 100` で算出する派生値（%）。保存せず、表示・分析時に都度計算する。
_Avoid_: 歩留まり、ロス率

### 素材

**Bean（生豆）**:
焙煎前の未加工コーヒー豆。産地・購入店・在庫量を管理する。焙煎済みの豆は指さない。同じ豆を買い直した場合、同一ロットなら既存 Bean の Stock を補充し、別ロットなら新規 Bean として登録するかはユーザーが判断する。アプリは制約しない。
- `name`: ユーザーがこの豆を呼ぶ名前（例: エチオピア イルガチェフェ）
- `origin`: 国・地域（例: エチオピア）
- `productName`: ショップの商品名・ロット名（例: イルガチェフェ G1 ナチュラル 2023クロップ）
_Avoid_: コーヒー豆、原料豆

**Stock（在庫）**:
特定の Bean が現在何グラム残っているかを示す数値。新規 RoastLog 保存時に `weightBeforeG` 分だけ自動デクリメントされる。編集・削除では変動しない。
_Avoid_: 残量、在庫量

**ImportedAt（輸入時期）**:
ショップが生豆を仕入れた時期。ショップのパッケージに記載された情報を転記する。豆の鮮度判断の参考指標。ユーザー自身の購入日（`purchasedAt`）とは別概念。
_Avoid_: 入荷日、仕入れ日

### 焙煎の評価

**RoastLevel（焙煎度）**:
焙煎後の実際の仕上がりを示すラベル（例: 浅煎り、中煎り）。目標や意図ではなく結果を記録する。機械の設定値でもない。ユーザーが名前・色・並び順を自由に定義できる。デフォルトは7段階プリセット。
_Avoid_: 焙煎レベル、ロースト度、目標焙煎度

**Crack（ハゼ）**:
焙煎中に豆が膨張して発する破裂音のイベント。1ハゼ（FirstCrack）と2ハゼ（SecondCrack）の2種類があり、それぞれ焙煎の進行度の指標となる。発生タイミングを秒数で記録する。
_Avoid_: クラック、爆ぜ

**Tasting（テイスティング）**:
焙煎後のコーヒーを飲んで行う官能評価。フレーバータグ（複数選択）と6軸の5段階評価で構成される。v1 は軸固定。
_Avoid_: 味覚評価、カッピング

**TastingAxis（テイスティング軸）**:
Tasting を構成する評価軸。v1 は Sweetness / Acidity / Body / Bitterness / Aftertaste / Cleanliness の6軸固定。
_Avoid_: 評価項目、評価軸

**Cleanliness（クリーンさ）**:
雑味のなさを示すテイスティング軸。焙煎ミス（焦げ・生焼け）の検出指標として機能する。値が2以下の RoastLog はログ一覧で警告色を表示する。
_Avoid_: クリーン、雑味スコア

**OverallScore（総合スコア）**:
1〜5の5段階で RoastLog 全体を評価するスコア。テイスティング6軸とは独立して入力する。
_Avoid_: 総合評価、レーティング

**FlavorTag（フレーバータグ）**:
テイスティングで感じた風味を表すユーザー定義ラベル（例: フローラル、チョコレート）。名前と色を持つ。複数選択可能。
_Avoid_: 風味タグ、フレーバーラベル

### レシピ管理

**BestRecipe（ベストレシピ）**:
ある Bean に対して「最も良い焙煎結果」として指定した1件の RoastLog。`Bean.bestLogId` で参照する。豆1種につき1件のみ。
_Avoid_: ベストログ、おすすめレシピ

**DiffSummary（差分サマリー）**:
同じ Bean の直前 RoastLog との比較パネル。「直前」は `roastDate` 降順で最も近い過去のログを指す。1ハゼ・2ハゼのタイミング差、重量減少率の差、総合スコアの差、気温の差を表示する。ログ詳細画面にのみ表示する。その Bean の最初の RoastLog では表示しない。
_Avoid_: 前回比較、差分パネル

### 環境・機器

**RoastDevice（焙煎機）**:
焙煎に使用する機器のマスターデータ。名前・方式・メモを持つ。デフォルトプリセットは「weroast HOME ROASTER」。
_Avoid_: 焙煎器、ロースター

**WeatherCondition（気象条件）**:
焙煎時の外気温・外気湿度・天気コード（WMO）の組み合わせ。Open-Meteo API から自動取得する。取得に失敗した場合は空欄のまま手動入力に委ねる。
_Avoid_: 天気、気象データ

**IndoorTemp（室内気温）**:
焙煎場所の室内温度。手動入力のみ。WeatherCondition の外気温とは独立して管理する。
_Avoid_: 室温、内気温

## Relationships

- **Bean** は複数の **RoastLog** を持つ
- **RoastLog** は1つの **Bean** に属する
- **RoastLog** は1つの **RoastLevel** を持つ
- **RoastLog** は0または1つの **Tasting** を持つ
- **Tasting** は複数の **FlavorTag** を参照する
- **Bean** は0または1つの **BestRecipe**（= RoastLog）を持つ
- **DiffSummary** は RoastLog 詳細画面で、同じ Bean の直前 RoastLog と比較して生成される

## Example dialogue

> **Dev:** 「同じ豆を2回連続で焼いたとき、どう記録しますか？」
> **Domain expert:** 「2件の **RoastLog** を作ります。**Batch** ごとに **Crack** のタイミングや **WeightLossRate** が変わるので、まとめて1件にすると振り返りが不正確になります。」

> **Dev:** 「**Stock** がマイナスになったらどうしますか？」
> **Domain expert:** 「警告は出しますが保存はできます。記録を省略した焙煎や豆を取り除いたときに **Stock** がずれることがあるので、手動で再調整します。**RoastLog** を編集・削除しても **Stock** は変わりません。」

> **Dev:** 「**BestRecipe** はどうやって決めますか？」
> **Domain expert:** 「手動で指定します。**OverallScore** が最高の RoastLog を候補として提示してくれますが、最終的には自分で選びます。豆1種につき1件だけ指定できます。」

## Flagged ambiguities

- 「焙煎記録」は **RoastLog** に統一。「セッション」は複数バッチを連想させるため使わない。
- 「コーヒー豆」は焙煎済みを含む曖昧な表現のため **Bean（生豆）** に統一。
- 「カッピング」はSCAプロトコルに基づく特定手法を指すため **Tasting** とは区別する。
